# Pangyo vs Cheongna Web GIS

판교 분석권역과 청라 분석권역을 비교하는 과제용 Web GIS입니다.

## 사용 데이터

- `analysis-boundaries.geojson`
- `census_merged.geojson`
- `subway_network.zip`
  - `nodes.tsv`
  - `links.tsv`
- 건축물대장 표제부 CSV
- 건축물대장 부속지번 CSV
- UD210 용도지역 SHP

## 기준연도

- 집계구 경계: `2025년 2분기`
- 건축물대장 CSV: 파일명 기준 `2026-06-18` 수집본
- subway_network: 현재 제공본
- UD210: 파일명 기준 `2026-06` 제공본

## 분석권역 정의

본 연구는 공식 사업구역 SHP를 확보하지 못했기 때문에, 판교테크노밸리와 청라국제업무지구의 중심 업무권역을 기준으로 분석권역을 정의하였다. 해당 경계는 법적 경계가 아니라 비교분석을 위한 분석 단위이며, 모든 수치는 이 분석권역 기준으로 산출하였다.

## 처리 과정

1. 집계구 경계와 인구, 가구, 종사자 데이터를 조인하여 `census_merged.geojson`을 생성했다.
2. `analysis-boundaries.geojson` 기준으로 판교/청라 권역을 나누고 집계구를 clip했다.
3. 각 권역의 총인구, 총가구수, 총종사자수, 직주비를 계산했다.
4. `subway_network.zip`의 `nodes.tsv`, `links.tsv`를 이용해 판교역과 청라국제도시역 기준 접근성을 계산했다.
5. 접근권은 실제 등시간권 폴리곤 대신 30분/60분 도달 가능 역 포인트로 시각화했다.
6. 건축물대장 표제부와 부속지번을 PNU 기준으로 매칭해 주용도 통계를 계산했다.
7. UD210은 분석권역 기준으로 clip을 시도했고, 결과를 `landuse_stats.json`에 기록했다.

## 한계점

- 공식 사업구역 SHP 미확보
- 분석권역 기준 비교 수행
- UD210 교차 객체 부재
- 건축물 도형 미확보
- 건축물 클릭 기능 미구현
- 접근권 시각화는 실제 등시간권 폴리곤이 아니라 도달 가능 역 포인트 표현

## 구현 범위

- 판교/청라 개별 지도 보기
- Split Map 비교 보기
- 권역별 총인구, 총가구수, 총종사자수, 직주비
- 10분~60분 접근성 수치
- 30분/60분 도달 가능 역 시각화
- 건축물 주용도별 건축물 수
- 건축물 주용도별 연면적 비율
- 평균 용적률 비교
